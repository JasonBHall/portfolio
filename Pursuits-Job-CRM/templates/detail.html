{# ----- Drag-and-drop wiring ------------------------------------------ Why we need JS here at all: HTML5 drag-and-drop fires DOM events with File objects in event.dataTransfer.files. Those Files aren't in the DOM, so HTMX's declarative model (which works by reading form values) can't see them. The minimal escape hatch: catch the drop event ourselves, build a FormData, and call htmx.ajax() to send it. HTMX still handles the response swap, so we get all the same target/indicator behavior as the picker path.
