{# Filter sidebar. Every checkbox is wired to GET / via HTMX. When any one toggles, the whole form serializes (its name=value pairs) and posts to the index route, which returns the rows fragment to swap into #opp-list-wrapper. Why a
wrapping all the checkboxes: HTMX serializes the closest enclosing form when you put hx-* on a nested input. So the sidebar form here lets each checkbox include EVERYTHING in the sidebar — status filters, age filters, source, tags, the show-snoozed toggle — in a single request. The hx-* attrs live on the form so all child inputs inherit them, but each checkbox carries hx-trigger="change" individually because forms don't auto-submit on change. (The tag's default behavior is "submit on Enter or button click" — change events need explicit triggers.) hx-include="[name='q']" pulls the search box into the request too, so filter+search compose. The search input is in the page header, not inside this form. #}
